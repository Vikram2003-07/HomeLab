### HomeLab Project

1. Installed kali,windows10,windows-server-2022(ADDC01),Ubuntu-server-22.0(Splunk)

2. Setting up Sysmon and Splunk

2.1 Setting up NAT Network in VirtualBox
    - go to network section in virtualbox
    - go to network tab and press create button
    - under 'General Options' tab set Name: AD-Project, ipv4-prefix: 192.168.10.0/24 and check the box Enable DHCP
    - click apply
    - now go to machines tab, for all our machines - kali,win10,ADDC01,Splunk, set network type as NAT Network make sure that name is AD-Project

2.2 Run Splunk Machine(Ubuntu-server)
    - check the ipv4 address, make sure it's in 192.168.10.x/24
    - now we need to set manual dhcp, so open netplan, i.e, run :sudo nano /etc/netplan/50-cloud-init.yaml
    - edit the file to - 
      network:
        version: 2
        ethernets:
            enp0s3:
            dhcp4: no
            addresses: [192.168.10.10/24]
            nameservers:
                addresses: [8.8.8.8]
            routes:
                - to: default
                via: 192.168.10.101
    
    - make sure the indentaion Levels
    - run :sudo netplan try, ignore the warnings, if prompted to press enter key, press enter key
    - if not prompted anything run :sudo netplan apply, ignore the warnings
    - now check the the ip address rub:ip a, it should show you ipv4 as 192.168.10.10, otherwise you need to configure netplan yaml file prperly from 2.2
    - if everything is correct run :ping google.com, it shoud ping properly.

2.3 Install Splunk Enterprise
    - Download the splunk enterprise version(Linux - .deb file) from the splunk website to your host machine.
    - install gest add-ons on Ubuntu server - sudo apt-get install virtualbox-guest-additions-iso -y
    - it will pop a purple screen just hit enter.
    - then add shared folder where you have installed splunk entireprice version(.deb file)
    - then reboot the ubuntu server
    - add user to vboxsf group - sudo adduser mydfir vboxsf
    - create a directory named share to mount our share folder.
    - run sudo mount -t vboxsf -o uid=1000,gid=1000 ActiveDirectory_HomeLab share/
    - then change directory to /opt/splunk/ and change the user to splunk, by typing sudo -u splunk bash
    - then change directory to bin, cd bin, then type - ./splunk start, then press y to enter.
    - now it will ask for administrator user name and password, set it as per your choice, then splunk instance will be running under http://127.0.0.1:8000
    - now type exit to come out of splunk user to mydfir, then type - cd bin and then type - sudo ./splunk enable boot-start -user splunk, this will start the splunk as a splunk user when ever reboot the server.

3.1 Install splunk forworder and sysmon in target machine(Windows10)
   - Rename windows10 to target-PC under rename this pc settings.
   - set the ip address as 192.168.10.100, subnet mask as 255.255.255.0 and default gateway 192.168.10.1 under Network and Internet > Change adapter > Ethernet > properties > Internet prtocol version 4.
   - now check the ipv4 address, it should be 192.168.10.100.
   - now open browser and navigate to http://192.168.10.10:8000 to access Splunk Web Interface.
   - install splunk forwarder in Windows10 from Splunk website using existing account.
   - while installing make sure the Receiving Server is set to 192.168.10.10 (Splunk Server IP).
   - now start installing sysmon and sysmon olag configuration on target-PC.
   - for sysmon configuration go to sysmon olaf configuration github, then go to sysmonconfig.xml file, download it.
   - now Open the Powershell as admin, Change the directory to where you have downloaded the Sysmon and configuration and then run the command > .\Sysmon64.exe -i ..\sysmonconfig.xml
   - now we need to specify what type of logs we want to forword to the splunk.
   - goto This Pc > C drive > program files > splunkuniversalforworder > etc > system > local, here you need to keep the file named inputs.conf, the content of this files should be:
   [WinEventLog://Application]
   index = endpoint
   disabled = false

   [WinEventLog://Security]
   index = endpoint
   disabled = false

   [WinEventLog://System]
   index = endpoint
   disabled = false

   [WinEventLog://Microsoft-Windows-Sysmon/Operational]
   index = endpoint
   disabled = false
   renderXml = true
   source = XmlWinEventLog:Microsoft-Windows-Sysmon/Operational
   - restart the splunk service.

3.2 Setting up Splunk server
    - go to the splunk dashboard use target-Pc web browser.
    - go to settings > indexes, click NewIndex, name it as endpoint and click save.
    - again go to settings > Forwarding and Receiving, click configure receiving then click on new receiving, enter 9997 for the portnumber and then click save.
    - now go to the search and reporting, type index=endpiont, you view the logs generated by target-PC.

3.3 setting up windows server
    - first rename the server as ADDC01 under settings.
    - install the splunk forworder same as the windows10.
    - check the hosts under splunk dashboard, now there should be 2 hosts, target-pc and ADDC01.
